# Project Template

## Directory Structure

Every new competition uses this structure:

```
competitions/{NN}_{name}/
├── config.yaml                 # Competition configuration (core!)
├── README.md                   # Competition notes + experiment results table
├── requirements.txt            # Competition-specific dependencies
│
├── data/
│   ├── raw/                    # Original data (.gitignore)
│   └── processed/              # Processed data (.gitignore)
│
├── notebooks/
│   └── eda.ipynb               # Exploratory analysis
│
├── scripts/
│   ├── run_r01_baseline.py     # Each experiment is one script
│   ├── run_r02_*.py            # Naming: R{NN}_{approach}
│   └── ...
│
├── outputs/
│   ├── submissions/            # Submission files
│   └── models/                 # Model files (.gitignore)
│
├── docs/
│   ├── experiment_log.md       # Structured experiment log
│   └── feature_catalog.md      # Feature catalog
│
└── mlruns/                     # MLflow experiment data (.gitignore)
```

## Naming Conventions

### Scripts
- Format: `run_r{NN}_{approach}.py`
- Examples: `run_r01_baseline.py`, `run_r06_crossjoin.py`, `run_r12_ensemble.py`
- Each script must have a docstring with "Changes from R{NN-1}" section

### Submissions
- Format: `submission_r{nn}_{approach}.csv`
- Auto-generated by `get_submission_filename()`

### Experiments
- MLflow run name matches script: `R06_crossjoin`
- Use `log_experiment(run_name="R06_crossjoin", ...)`

## Setup Checklist

For each new competition:

1. [ ] Create directory from template
2. [ ] Fill in `config.yaml` (see config-spec.md)
3. [ ] Download data to `data/raw/`
4. [ ] Add shared src to Python path (template scripts handle this)
5. [ ] Initialize MLflow: `mlflow db upgrade sqlite:///mlflow.db` (first time only)
6. [ ] Write `README.md` with competition overview
7. [ ] Create `notebooks/eda.ipynb`
8. [ ] Write `scripts/run_r01_baseline.py` from template
