## CV-LB Gap 诊断（关键 lesson）

| Stacker | OOF | LB | Gap | 解读 |
|---|---|---|---|---|
| cdeotte_lr_stacker | 0.9703 | 0.97101 | **-0.0007** | 几乎无 gap，已收敛 |
| 自训 8-source LR | 0.9724 | 0.9626 | **+0.0098** ⚠️ | 严重 OOF 过拟合 |
| 自训 9-source LGB | 0.9632 | 0.9625 | +0.0007 | 偶然一致 |

**Insight**: OOF 高 ≠ LB 高。自训练 stacker 在 OOF features 上过拟合了（OOF accuracy 0.9724 vs cdeotte 0.9703 是 OVERFITTING 信号）。cdeotte 的 stacker 用 GPU + 8 个独立 base 训练，OOF 没有过拟合，所以 OOF↔LB 一致。

## 关键洞察

1. **cdeotte_lr_stacker 单模型 > 任何自训练 blend/stacker**（LB 0.97101 vs 自训最高 0.97021）
2. **OOF 高 ≠ LB 高**（自训 8-source LR OOF 0.9724 vs LB 0.9626 — 0.0098 过拟合 gap）
3. **schema8 单模型 > 任何 blend**（OOF 0.9668 vs best blend 0.9637）
4. **AutoGluon-first 在 S6E6 不如 OOF 最强单模型**（0.9630 vs 0.9668）
5. **GSD verifier 抓到了 format trap**（predict_proba vs predict for accuracy）
6. **9 天内大幅突破很难** — 0.97101 多次 reproduce 表明接近 public LB ceiling
